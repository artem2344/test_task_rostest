
# Тестовое задание: Фильтр врачей (WP)

## Установка

### Вариант 1: Сборка на базе Underscores
1. Создайте пустую заготовку темы на [underscores.me](https://underscores.me).
2. Установите полученную тему на WordPress.
3. Скопируйте в папку темы файлы из директории `custom_files`.
4. В файле `{тема}/functions.php` добавьте следующую строку:
   `require get_template_directory() . '/inc/custom-function.php';`
5. В админ-панели WP перейдите в **Настройки -> Постоянные ссылки** (`/wp-admin/options-permalink.php`) и нажмите кнопку **«Сохранить изменения»**.

### Вариант 2: Установка готовой темы
1. Скачайте архив `full_theme.zip`.
2. Распакуйте архив в директорию `wp-content/themes/`.
3. Активируйте тему **alarioshin_test_rostest** в административной панели.
4. Перейдите в **Настройки -> Постоянные ссылки** и нажмите **«Сохранить изменения»**.

## Администрирование контента

Управление контентом осуществляется через боковое меню WordPress:

1. **Доктора (Custom Post Type):** Управление карточками врачей — `/wp-admin/edit.php?post_type=doctors`
2. **Специализации (Taxonomy):** Настройка специальностей — `/wp-admin/edit-tags.php?taxonomy=specialization&post_type=doctors`
3. **Города (Taxonomy):** Управление городами присутствия — `/wp-admin/edit-tags.php?taxonomy=city&post_type=doctors`

## Структура файлов

В ходе разработки были добавлены или изменены следующие файлы:

*   `archive-doctors.php` — основной шаблон архива (вывод фильтров и карточек).
*   `single-doctors.php` — шаблон детальной страницы записи.
*   `inc/custom-function.php` — кастомные функции регистрации типов записей и логики фильтрации.
*   `js/script.js` — скрипты для работы фронтенд-части.
*   `style.css` — стили оформления карточек, пагинации и состояний фильтра.

## Особенности реализации

1.  **Логика фильтрации:**
    *   Между разными таксономиями (Город + Специализация) применяется логика **«И»**.
    *   Между значениями внутри одной таксономии применяется логика **«ИЛИ»**.
2.  **Структура URL:**
    Значения таксономий в URL передаются на кириллице. Это гарантирует, что фильтрованные страницы будут доступны по прямой ссылке даже при изменении ID терминов в базе данных (например, `?specialization=Глаза` вместо `?specialization=5`).
3.  **Таксономия «Город»:**
    Выбрана неиерархическая структура (`non-hierarchical`), так как для городов не требуются дочерние объекты.
